Support and beta testing

Samsara is free, open source, and still in beta. A clear bug report from the machine where something failed is the most useful contribution you can make.

Report a problem

Use the public GitHub issue tracker. Include what you expected, what happened, the Samsara version, and the smallest set of steps that reproduces it.

Open a GitHub issue

Attach the useful evidence

  • Open the tray menu and choose Live Log.
  • Reproduce the problem once.
  • Copy the relevant final lines from %USERPROFILE%\.samsara\logs\samsara.log.
  • For a visual problem, include a screenshot and your display scaling percentage.

Review logs before posting: dictated text, app names, file paths, or provider errors may contain information you do not want to publish.

Before reporting

Check the current documentation and the existing issue list. For CUDA problems, confirm that all ten files from the verified runtime pack were extracted and that Samsara was restarted.

Want to beta test?

Install the newest release, use it in a real workflow, and report friction—not only crashes. Hands-free navigation, long dictation, different microphones, multi-monitor Show Numbers, and high-DPI displays are especially valuable test areas.

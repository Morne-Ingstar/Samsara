Samsara Voice Dictation - Changelog
====================================

2026-02-04
----------

### Critical Bug Fix
- Clipboard Preservation Fixed on 64-bit Windows
  - Root cause: Windows API handles (64-bit pointers) were overflowing because ctypes was treating them as 32-bit integers
  - `save_clipboard()` was silently returning empty dictionaries
  - User's clipboard was permanently replaced with dictated text after every paste
  - Fix: Proper Windows API type definitions (HANDLE = c_void_p, SIZE_T = c_size_t, etc.)

### Code Improvements
- New centralized `samsara/clipboard.py` module
  - Single shared clipboard lock between dictation.py and samsara/commands.py
  - Eliminates race conditions from duplicate lock objects
  - Comprehensive format preservation (text, images, files, HTML, etc.)
  - Proper error handling for locked clipboard scenarios

- Multi-Format Sound Support
  - Now supports MP3, OGG, FLAC, M4A in addition to WAV
  - Requires pydub library and ffmpeg for non-WAV formats
  - Automatic format detection - just drop files in sounds/ folder
  - Falls back to WAV-only if pydub not installed

- Sound Theme System

  - Theme selector in Settings > Sounds tab
  - Easy one-click theme switching with preview
  - Themes stored in sounds/themes/ folder
  - Create custom themes by adding new theme folders

- Audio Quality Improvements
  - Fixed start sound not playing (winsound conflict with InputStream)
  - Added 250ms delay before mic input to let sound complete
  - High-quality sound generation with proper fade in/out
  - Silence padding at end of sounds to prevent cutoff artifacts

### Wake Word Improvements
- Faster Response Time
  - Reduced silence detection threshold for quicker processing
  - Configurable wake_detection_silence in config (default 1.2s)
  - Separate wake_command_timeout for post-wake-word listening (5s)

- Better Speech Capture
  - Fixed issue where speech after wake word was cut off
  - Increased silence threshold to capture natural "wake word [pause] command" speech
  - Added garbage filter to ignore noise/hallucinations (punctuation-only output)

### Settings Persistence Fix
- Mode changes in Settings now properly take effect
- Stops old mode and starts new mode when Apply is clicked
- Runtime state now syncs with saved config

### Exit/Quit Fix
- Exit from system tray menu now works reliably
- Uses os._exit() to avoid tkinter thread conflicts
- Properly stops all active modes before exit

### New Dependencies
- pydub (optional) - for MP3/OGG/FLAC support
- ffmpeg (system) - required by pydub for audio decoding


2026-01-23
----------

### Wake Word Dictation Enhancements
- Pause Word Support (Fully Implemented)
  - Say pause word during dictation to reset the silence timer
  - Allows thinking mid-dictation without triggering timeout
  - Pause word stripped from output, remaining text preserved
  - Configurable phrase: pause, hold on, wait (custom supported)
  - Disabled by default, enable in Settings > Advanced

- Dictation Auto-Finalization
  - For dictate/short dictate modes: auto-outputs text after silence timeout
  - Uses a finalization timer that restarts on each new speech chunk
  - Long dictate mode unaffected (still requires end word)

- Wake Word Debug Window Improvements
  - Timer display showing countdown to timeout (color-coded green to red)
  - Flow indicator showing current step in the wake word pipeline
  - Mode selector dropdown to force into a dictation mode for testing
  - Simulate buttons for end word, cancel word, and pause word
  - Timer resets properly on pause word and new speech

### Performance
- Audio Cue Latency Fix (Persistent Stream)
  - Persistent audio output stream stays open for the lifetime of the app
  - Sound playback requests queued to a dedicated worker thread
  - Eliminates device re-acquisition delay that worsened after idle/background
  - All sounds normalized to 44100 Hz mono at load time (no stream recreation)
  - Stream auto-recovers if audio device disconnects or errors

### Bug Fixes
- Clipboard Preservation (Full Format Support)
  - Uses Windows clipboard API (ctypes) to save/restore ALL clipboard formats
  - Preserves images, files, rich text, HTML - not just plain text
  - Applies to both dictation output and voice command text insertion
  - Previous pyperclip-based approach only preserved plain text strings
